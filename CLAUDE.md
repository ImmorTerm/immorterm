# ImmorTerm

Persistent, AI-native terminal sessions. A background daemon keeps terminal sessions
alive across crashes and restarts; the rendering core is GPU-accelerated (Rust + WASM);
AI agents can draw inline HTML/diagrams; and a companion binary gives local-first memory.

## Component map

| Component | Language | Location | Purpose |
|-----------|----------|----------|---------|
| VS Code extension | TypeScript | `apps/extension/` | Terminal lifecycle, status bar, AI session tracking |
| Desktop app | Rust (Tauri) | `apps/immorterm-app/` | Standalone GPU terminal (own Cargo lockfile) |
| Terminal engine | Rust + WASM | `apps/immorterm-ai/` | GPU render core, session daemon, WebGPU, AI overlays |
| CLI | TypeScript | `apps/immorterm/` | Setup, service management, health checks |
| Docs site | TypeScript | `apps/docs/` | Documentation site (Next.js + Fumadocs) |
| Shared libs | TypeScript + Rust | `libs/*` | Also published as `@immorterm/*` on npm |
| opencode plugin | TypeScript | `packages/immorterm-opencode-plugin/` | opencode → ImmorTerm hook bridge |

## Build commands

```bash
bun install                       # all TS workspace deps

# TypeScript
bun run build:cli                 # CLI
bun run build:extension           # VS Code extension (tsc + esbuild)
bun run build:docs                # docs site
bun run test:cli                  # CLI tests + hook tests
bun run test:extension            # extension tests

# Rust (immorterm-ai engine)
cargo check                       # native crates
bun run build:immorterm-ai        # full engine + WASM build + local deploy
bun run build:immorterm-ai:quick  # WASM + deploy only
bun run test:immorterm-ai         # core + render tests
bun run lint:immorterm-ai         # clippy (native)
bun run lint:immorterm-ai:wasm    # clippy (wasm32 target)

# Desktop app (standalone Cargo workspace)
cd apps/immorterm-app/src-tauri && cargo check
```

## Layout notes

- The Rust workspace root is `Cargo.toml`. The Tauri app (`apps/immorterm-app/src-tauri`)
  is a standalone crate with its own lockfile, excluded from the workspace.
- The WASM crate builds with `--target wasm32-unknown-unknown` and is excluded from
  default cargo builds. `scripts/build-immorterm-ai.sh` runs the full WASM pipeline.
- Hook templates are generated by the installer in `libs/services/` (source of truth);
  the extension's `apps/extension/src/services/memory/hook-installer.ts` is thin glue.
- The companion memory engine and MCP gateway are separate binaries/repos; the CLI
  downloads their release binaries during `immorterm setup`.
